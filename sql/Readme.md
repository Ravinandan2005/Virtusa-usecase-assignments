# SQL Use Case: Retail Insights

## Overview

Retail Insights is a comprehensive database solution designed to help retail store managers analyze inventory, sales performance, and product lifecycle management. The system provides critical business intelligence by identifying problematic inventory situations, revenue patterns, and stock risk classifications.

## Business Context

Retail stores struggle with inventory management challenges such as overstocking, dead stock accumulation, and expiring products. These issues result in financial losses and operational inefficiencies. A data-driven approach using a well-designed relational database can provide actionable insights to improve inventory decisions and maximize revenue.

## Problem Statement

The retail store system addresses these challenges:
- Identify products that are nearing expiration within the next 7 days
- Detect products that have not been sold recently (dead stock, 60+ days)
- Analyze revenue contribution by product category
- Classify products by risk level based on expiry and stock status

## Database Design

### Tables

**categories** - Product categories
- category_id (Primary Key)
- category_name (String, Unique)

**products** - Product details
- product_id (Primary Key)
- product_name (String)
- category_id (Foreign Key references categories)
- price (Decimal)
- stock_quantity (Integer)
- expiry_date (Date)

**sales_transactions** - Sales history
- transaction_id (Primary Key)
- product_id (Foreign Key references products)
- quantity_sold (Integer)
- sale_amount (Decimal)
- transaction_date (Date)

### Relationships
- One category can have many products (1:N)
- One product can have many sales transactions (1:N)

## Key Queries Implemented

### 1. Expiring Products

Identifies products expiring within the next 7 days with stock quantity above zero. Helps prioritize sales promotions and prevent loss.

Query logic: Filter products where expiry_date is between today and 7 days from today.

### 2. Dead Stock Analysis

Detects products not sold in the last 60 days. These items occupy valuable shelf space and represent tied-up capital.

Query logic: Find products with no sales transactions in the past 60 days.

### 3. Revenue Analysis by Category

Calculates total revenue generated by each product category. Essential for understanding which categories drive business value.

Query logic: Sum all sales_amount grouped by category with category names displayed.

### 4. Stock Risk Classification

Classifies each product as HIGH, MEDIUM, or LOW risk based on:
- HIGH: Expiring within 7 days with stock > 50
- MEDIUM: Expiring within 30 days
- LOW: All other products

Helps prioritize management actions and prevent losses.

## How to Run

### Step 1: Create Database
```sql
CREATE DATABASE retail_insights;
\c retail_insights
```

### Step 2: Execute Script
Run the retail-insights.sql file which contains:
- CREATE TABLE statements
- Sample data insertion (INSERT statements)
- All analytical queries

```bash
psql -U postgres -d retail_insights -f retail-insights.sql
```

### Step 3: View Results
Execute individual queries to see analysis:
```sql
SELECT * FROM expiring_products;
SELECT * FROM dead_stock;
SELECT * FROM revenue_by_category;
SELECT * FROM stock_risk_classification;
```
